分支​

git 存储文件时，每一次代码提交都会创建一个与之对应的节点，git 就是通过一个一个的节点来记录代码的状态。节点会构成一个树状结构，树状结构就意味着，这个树就会存在分支，默认情况下仓库只会有一个分支命名为 master

在使用 git时我们可以创建多个分支，分支与分支之间相互独立，在一个分支上修改代码时不会影响其他的分支。

git branch ## 查看分支
git branch <branch name> ## 创建分支 
git branch -d <branch name> ## 删除分支

git switch <branch name> ## 切换分支
git switch -c <branch name> ## 创建并切换分支

git merge <branch name> ## 合并分支

在开发中都是在自己的分支上编写代码，代码编写完毕后，再将自己的分支合并到主分支中

变基（rebase）​

在开发中除了通过 merage来合并分支外，还可以通过变基来完成分支 的合并。

通过 merge合并分支时，在提交记录中会将所有的分支创建和分支合并的过程全部显示出来，这样当项目比较复杂，开发过程比较波折时，我们必须要反复的创建、合并、删除分支。

这样一来将会使得代码的提交记录变的极为混乱。

原理（变基时发生了什么）：

  1. 当我们发起变基，git会找到两条分支最近的共同祖先
  2. 对比当前分支相对于祖先的历史提交，并且将他们提取出来存储到一个临时文件中
  3. 将当前部分指向目标的基底
  4. 以当前基地开始，重新执行历史操作

变基和 merge 对于合并分支来说最终的结果是一样的！但是变基会使得我们的代码提交记录更整洁更清晰！注意大部分情况下合并和 变基 是可以互换的，但是如果分支已经提交远程仓库，那么尽量不要使用 变基

git rebase <branch name> # 变基

远程仓库（remote）​

目前我们对于 git 所有操作都是在本地进行的。但是在开发中显然不是这样的，这时候我们就需要一个远程的 git 仓库。

远程的 git 仓库和本地的仓库没有本质的区别，不同点是在于远程的仓库可以被多人同时访问使用，方便多人协作开发。在实际工作中，git 的服务器通常由公司搭建内部使用或是购买一些公共的的私有 git 服务器。学习阶段，直接使用一些开放的公共的 git 仓库。

目前常用的库有两个：GitHub 和 Gitee

将本地库上传到 GitHub

# git remote add <remote name> <url>
git remote add origin https://github.com/3320753238/git-demo.git

# 修改分支的名字为 main
git branch -M main

# git push 将代码上传致服务器
git push -u origin main

将本地库上传 Gitee

git remote add gitee https://gitee.com/arj12345/git-demo.git
git push -u gitee main

远程库的操作的命令​

git remote # 列出当前关联的远程库
git remote add <远程库名> <url> # 关联远程仓库
git remote remove <远程库名> # 切断远程关联的仓库
git push -u <远程库名> <分支名> # 向远程仓库托送代码，并和当前分支关联
git push <远程库名> <本地分支名>:<远程分支>
git clone <url> # 从远程库下载代码

git push   # 如果本地库的版本低于远程库 push 默认推送不上去
git fetch  # 要想推送成功，必须先确保本地库和远程库的版本一致，fetch 它会从远程库下载所有代码，但是它不会将代码和当前分支自动合并
		   # 使用fetch 拉取代码，必须要手动对代码进行合并

git pull   # 服务器拉取代码并合并

注意：推动代码前，一定要在远程库中拉取新的代码

tag 标签​

当头指针没有指向分支的头部时，这种状态我们称为分离头指针（HEAD detached）在分离头指针的状态下也可以操作代码，但是这种状态下进行操作不会出现在任何分支上（所以不要在分离头指针的状态下操作代码仓）

如果一定要回到后边的节点进行操作，则可以选在创建分支后再操作

git switch -c <分支名> <提交的节点id>

标签：可以为提交记录设置标签，设置标签以后可以通过标签快速的识别开发节点

git tag
git tag 版本
git tag 版本 提交id

# 推送标签到远程
git push 远程仓库 标签名
git push 远程仓库 -tags # 推送所有
# 删除标签
git tag -d 标签名 # 删除标签
git push 远程仓库 --delete 标签名 # 删除远程标签

gitignore​

默认情况下，git会监视项目中所有内容，但是有些内容比如 node_modules目录中的内容我们不希望它受 git 管理。

我们可以再项目目录中添加一个 .gitignore文件设置git忽略的内容

node_modules
yarn.lock
# 通配所有 .log 扩展名的文件
*.log

github的静态界面​

在 github中，可以将自己的静态页面直接部署到 github上，它会提供一个地址，使得我们的网页可以在互联网上访问

要求：

  • 静态页面的分支必须叫：gh-pages
  • 如果希望页面可以通过 xxxxx.github.io 访问则需要将库的名字配置为 xxxx.github.io
